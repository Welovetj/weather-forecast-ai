# 🌤️ Weather Forecast AI

A full-stack AI-powered weather forecast application built for the **PM Accelerator AI Engineer Internship** technical assessment.

It combines real-time weather data from [OpenWeatherMap](https://openweathermap.org/) with natural-language insights generated by **GPT-4o-mini**, served through a **FastAPI** backend and a **React + TypeScript** frontend.

---

## ✨ Features

| Feature | Detail |
|---|---|
| 🌍 Global city search | Search any city worldwide with optional country code |
| 🌡️ Current conditions | Temperature, feels-like, humidity, wind, pressure, visibility, sunrise/sunset |
| 📅 5-day forecast | Daily high/low, rain probability, wind, and weather condition |
| 📊 Temperature chart | Interactive area chart of the week's high/low trend (Recharts) |
| 🤖 AI insights | GPT-4o-mini summary, key highlights, and practical recommendation |
| 🔄 Unit toggle | Metric (°C), Imperial (°F), or Standard (K) |
| ⚡ Fallback AI | Rule-based insights when OpenAI key is absent |

## 🚀 Quick Start

### Prerequisites

| Tool | Version |
|---|---|
| Python | 3.12+ |
| Node.js | 20+ |
| (Optional) Docker | 24+ |

### 1. Clone & configure

```bash
git clone https://github.com/Welovetj/weather-forecast-ai.git
cd weather-forecast-ai

# Copy and fill in your API keys
cp backend/.env.example backend/.env
```

Edit `backend/.env`:

```env
OPENWEATHER_API_KEY=your_key_here   # https://openweathermap.org/api
OPENAI_API_KEY=your_key_here        # https://platform.openai.com  (optional)
```

> **Note:** The app works without an OpenAI key — it falls back to rule-based insights.

---

### 2a. Run with Docker (recommended)

```bash
docker compose up --build
```

- Frontend → http://localhost:3000  
- Backend API → http://localhost:8000  
- Swagger docs → http://localhost:8000/docs

---

### 2b. Run manually

**Backend:**

```bash
cd backend
python -m venv .venv && source .venv/bin/activate   # Windows: .venv\Scripts\activate
pip install -r requirements.txt
uvicorn app.main:app --reload
```

**Frontend:**

```bash
cd frontend
npm install --legacy-peer-deps
npm run dev
```

Open http://localhost:5173

## 📡 API Reference

Base URL: `http://localhost:8000`

| Method | Endpoint | Description |
|---|---|---|
| GET | `/` | API info |
| GET | `/health` | Health check |
| GET | `/api/weather/current?city=London&units=metric` | Current weather |
| GET | `/api/weather/forecast?city=London&units=metric` | 5-day forecast |
| GET | `/api/weather/insights?city=London&units=metric` | Current + forecast + AI insights |

Interactive docs: http://localhost:8000/docs

## 🔧 Tech Stack

**Backend**
- [FastAPI](https://fastapi.tiangolo.com/) — async Python web framework
- [Pydantic v2](https://docs.pydantic.dev/) — data validation & settings
- [httpx](https://www.python-httpx.org/) — async HTTP client
- [OpenAI Python SDK](https://github.com/openai/openai-python) — GPT-4o-mini integration

**Frontend**
- [React 18](https://react.dev/) + [TypeScript](https://www.typescriptlang.org/)
- [Vite 8](https://vite.dev/) — fast build tool
- [Tailwind CSS v4](https://tailwindcss.com/) — utility-first styling
- [Recharts](https://recharts.org/) — temperature trend chart
- [Lucide React](https://lucide.dev/) — icon set
- [Axios](https://axios-http.com/) — HTTP client

**Testing**
- [pytest](https://pytest.org/) + [pytest-asyncio](https://github.com/pytest-dev/pytest-asyncio)
- [Vitest](https://vitest.dev/) + [Testing Library](https://testing-library.com/)
